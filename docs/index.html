適用情境

設備對主系統交付

每台設備獨立維護發送 IO 與接收 IO，讓設備端與主系統端的訊號責任清楚分工。

位址衝突檢查

跨設備重複 IP 與主系統點位重複會即時提示，降低試車階段的追線成本。

專案文件產出

可匯出 Excel，一設備一 Sheet，方便給 PLC、SCADA、機電與現場人員共同確認。

核心功能

智慧位址填充

支援 DM、MR、R 暫存器等位址遞增，包含基恩斯 R015 到 R100 的 16 位元進位規則。

主系統視角

把所有完整 IO 行彙整成以主系統點位排序的反向對照表，快速找出訊號來源與方向。

桌面版 PLC 監控

Tauri 版本可讀取 KEYENCE KV Upper-Link 與三菱 SLMP / MC Protocol，顯示即時 IO 值。

使用流程

建立設備新增設備、設定 IP / Port、依現場設備順序拖曳排序。
填寫 IO 表輸入設備點位、訊號名稱、資料類型與主系統點位。
檢查衝突透過完整度、重複位址、主系統視角確認資料品質。
匯出交付將整理完成的對照表匯出成 Excel 或 JSON 專案檔。
